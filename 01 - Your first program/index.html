01 - Your first program

Time to get your feet wet with some actual programming! Although JavaScript is a general purpose programming languages and can be used in diffirent environments (inside or outside the web browser), we will focus on the web browser. We will introduce the developer console, which is going to be your best ally out there.

The developer console

Most major desktop browser integrates a tool called developer console, which allows to interact with a web page in different way, and even to type and execute JavaScript code. Different browsers have different key-shortcuts to access this tool (and some even call it with different names), so you should ask your search engine of choice how to access it on your browser. For example, Google Chrome users on GNU/Linux and Windows will find an Inspect command when right-clicking on a page, or can use the sortcut ctrl + shift + j to direct access the tool.

The console allow you to type and execute JavaScript code: try to access the console and ask it to oo some very easy calculations (output is reported for sake of examples): 

> 5
<- 5

> 5 + 9
<- 14

> 5 + &
x Uncaught SyntaxError: Unexpected token &
You will see a line like the last one (too) often: it means there is a syntax error in the code. In words, you wrote an invalid piece of JavaScript which broke the program, and the console is reporting this to you. The console also tries to inform you about file, line and nature of the error, although identifying an error is not as easy at it may seem.

Hello, World!

Examples so far were about computing some result, but we may also want to display. Usually, when you are using JavaScript in a web site, your result will be displayed inside the HTML entities, hence you will have to access them, compute what you need to compute, format in in HTML, append it to the document and so on. This is a reasonable procedure, but it depends strictly on the nature of the computation you are performing and on what you need to accomplish.

The developer console provides a general way to output information to deverlopers, which is the console.log function (technically speaking, console is an object and log is one field of its which is of type function). This allows to display on the console and inspect various data: 

console.log("Hello, World!");
Will print the string Hello, World! on the console. But it is more magican than that! It will use a different "layout" depending on the type of the object it is printing. For instance, an array containing two numbers, a string and a function will be rendered as: 
[1, 2, "my string", function]
