00 - Introduction

This is additional, optional material for students of the Responsive Web Design course. You are free to use and redistribute files and source code form this material for any use.

Material is organized as follows:

  • main folder contains subfolder named as "<number> - <name>", where numberare progressive numbers suggesting the order in which you should approac the exercises, while name is an explicative name of the content
  • each subfolder contains at least an index.html file, which explains the subject of the exercise, shows some examples and/or describes what to to in the exercises
  • each subfolder may contain "exercise files" named as <number>->exercise number>_<name>.html (for example: 99-58_some-exercise.html), where number is the same number identifying the subfolder, exercise are progressive number identifying exercises within a subfolder and name is mnemonic name for the exercise
  • every time an "exercise file" is provided, a proposal of "solution file" is also given, and it is named as the corresponding "exercise file" plus _solution appended to the name (example: 99-58_some-exercise_solution.html)
as a rule of thumb, always open the index.html file first, as it contains every info you need for a particular set of exercises.

Exercises are in the form of partially compiled HTML (and JavaScript) files which students can complete. The source code is richly commented in order to give precise instructions about what to do.

These exercises are not part of the course, hence they are not required to complete the course, and are not subjected to the peer review mechanism, they are completely optional. Nevertheless, students are encouraged to discuss about exercises in the course discussion forum.

What is this?

This file contains a base introduction about JavaScript, explaining some core concepts and introducing some jargon and best practices. You can use this file as a quick reference to JavaScript syntax. Since this is an introductory file, there are no exercises associated to this unit.

Comments

First thing you must learn are comments. Comments are a great way to tell what's going on and to annotate the code: 

// This is  comment on one line (inline)

/* This is a comment
   which may span
   over multiple lines (multiline) */
Comments are ignored by the JavaScript interpreter. Comments are for programmers, not for computers. As shown in the example, we have two possible syntax for comments, inline and multiline. Which one to use is completely up to you!

Indentation and names

Suppose you know nothing about JavaScript.
Now read those two pieces of code, which perform exactly the same task: 

// First code

// Second code
var x = 5;
var c = 4 * x;
Which one do you like best?

Here is another example: 

// First code
var a = 5;
			var b = 10;
	var c = a + 	b;
		var d =     c * c;

// Second code
var a = 5;
var b = 10;

The take home lesson: do use meaningful names, and try to properly align (jargon: indent) things.
Later we will give the correct names to these concepts. We will talk of "identifiers" rather than "names", and of "blocks" instead of "things". No matter how we will call them, you should apply some common sense when you write your code.

Variables and expressions

Terms like variable and expression recur often in programming. To make the long story short, a variable is a container for some data, and an expression is a calculation. Suppose you want to implement a web page which acts as a calculator, and you whish to provide a "compute area of a rectangle" feature: you need to know the length of the base and the height, then multiply them: 

var base_length = 9;
var height = 48;

var area = base_length * height;
base_height, height and area are variables, they are used to store data (numeric data, in this case, but it could be of any type). In the previous code, we have three assignment: some value is assigned to our variables. First two assignments are "easy", as only a constant, numeric value is used. Last line assigns the result of an expression to a variable (in this case the expression is multiplication between numbers, which are contained in the variables).

The act of "introducing" a variable is calling declaring a variable, the act of assigning it its first value is calling initialization: 

var triangle_base;        // This is only declared, it has no value
var triangle_angle = 45;  // This is declared AND initialized
side = 18;                // Initialized by NOT declared, see notes
It is possible to assign a value to a variable without declaring it with var. In this case, the variable becomes global (a concept not yet defined at this point): you are suggested to avoid this, unless you know what you are doing and have a good reason to do so.

Functions

Functions in JavaScript are very similar to mathematical partial functions, and are a core concept in JavaScript. On the practical side, suppose to have a web page to compute a long arithmetic expression: 

...
var result = (a * x * x * x) + (b * x * x) + (c * x) + d;
Now, suppose to do the same, but for different combinations of values of a, b, c, d, x: 
// Bad code!
...
var result1 = (10 * 5 * 5 * 5) + (7 * 5 * 5) + (8 * 5) + 4;
var result2 = (12 * 3 * 3 * 3) + (4 * 3 * 3) + (18 * 3) - 12;
...
var resultN = (20 * 2 * 2 * 2) + (2 * 2 * 2) + (57 * 2) + 9;
Writing the code as such is long and error-prone: it is possible to abstract the computation of the expression in a function: 
// Better code
function evaluation_of_expression(a, b, c, d, x) {
	return (a * x * x * x) + (b * x * x) + (c * x) + d;
}

var result1 = evaluation_of_expression(10, 7, 8, 4, 5);
var result2 = evaluation_of_expression(12, 4, 18, -12, 3);
...
var resultN = evaluation_of_expression(20, 2, 57, 9, 2);
Functions takes an arbitrary number of arguments (even 0) and may return a result. In given example, function takes four arguments and sums/multiplies them to compute a result. Functions defined in this way have a name, which allows them to be referenced and called. Anonymous functions (i.e. functions without a name) also exist, and we will discuss about them later in the exercises.

Functions have the purposes of abstracting a behavior, for instance the concept of area of a rectangle does not depend on a specific rectangle, it is defined for every rectangle and it is equal to the product between length of its base and its height: 

function rectangle_area(base_length, height) {
	var area = base_length * height;

	return area;
}
The piece of code inside a function, which is called body of the function can contain arbitrary code, it is not limited in term of length or complexity.

JavaScript is (also) a functional programming languages, which means functions are also values, hence they can be assigned to variables in the usual way: 

var area_function = rectangle_area;

// This is an anonymous function:
var my_function = function triangle_area(base, height) {
	return base * height / 2;
}
as a corollary, functions can be passed (and returned) to other functions. This concept is called functional in mathematics and higher order function in computer science, and has a number of teoretical and practical consequences.

Iterations

A concept you can easily think of is "do this thing 10 times, then do something else", which is often modeled as a for loop: 

for (var i = 0; i < 10; ++i) {
	do_this_thing();
}
do_something_else();
Reading this code is straightfoward. Let's have an index variable i, for i ranging from 0 (included) to 10 (excluded, hence 9), do something (in the example: call the do_something function), incrementing i by 1 each time. Observe that ++i is an alternative way to write i = i + 1

Another similar concept is: "As long as this condition is true, do this thing": 

while (is_some_condition_true()) {
	do_this_thing();
}
In this example, is_this_condition_true is a function which returns either true or false.

These concepts are called iterations.

Arrays and objects

So far we so only simple, numeric variables. JavaScript variables can hold different types of values, for instance strings (informally: text) or boolean values (true or false): 

var my_string = "This is a string!";
var my_bool1 = true;
var my_bool2 = false;
var my_bool3 = 3 < x;  // This one will be true is 3 < x, false otherwise
All of these are scalar type. You can read this informally as "simple" types, or "not-composed" type. We also saw that functions can be assigned to variables, that is a function type.

There are other two common types, which are arrays and objects. The former are an ordered collection of values (of any type), for instance a list: 

var my_array = new Array(5, 6, 1, "some string", new Array(), 3);
var second_element = my_array[1];
As shown in the example, an array can contain literally everything: numbers, strings, other arrays, functions, objects and so on. To access an element in an array we use its position between square brackets, or index. Array indexes start as 0, so first element is indexed as 0, second as 1 and so on.

Objects, instead, are records with labelled fields: 

var my_object = {
	name: "Guy",
	age: 28,
	friends: new Array("Mark", "Cindy")
};

var name = my_object.name;
formally, these are conjuctive data structures. As with arrays, objects can contains everything, including arrays and other objects.

Note for those who have experience with other object oriented programming languages: JavaScript objects of this type have almost nothing to do with objects and classes in object oriented programming. JavaScript can still be considered an object oriented language, but it uses different mechanisms to implement objects, inheritance and subtyping (namely: functions and prototypes).
