02 - Functions

JavaScript is a functional programming language, and functions play a vital role in it. For this set of exercise you will be given some functions and will be asked to implement others.

Defining a function

As we saw in the introduction, there are several way to define a function. Let's consider the "explicit" one: 

function my_function(x, y) {
	var z = x * y + x - y;

	return z - 5;
}
Definition of a function has the form: function <name> ( <argument list< ) { <body> }. Observe that spaces and newlines are not necessary, but are used to improve readability by programmers. The name can be any identifier, the argument list is a comma-separated list of arguments (for example: arg1, arg2, arg3) of any length (even 0), and the body is an arbitrary piece of JavaScript code.

Variables declared in the argument list must not be declared again in the function body: 

// This is wrong!
function foo(a, b) {
	var a = a;
	var b = b;

	return a + b;
}

// This is right.
function foo(a, b) {
	return a + b;
}

Examples

So, what can you do with functions? Here are some examples: 

/**
 * Prints an error message on the console.
 * @param message Message to print
 */
function display_error(message) {
	console.log("[ERROR] " + message);
}


/**
 * Prints a warning message on the console.
 * @param message Message to print
 */
function display_warning(message) {
	console.log("[WARNING] " + message);
}


/**
 * Tells whether an array is empty (i.e. contains no element).
 * @param array Array to examine
 * @return true if array contains no element, false otherwise
 */
function is_empty_array(array) {
	var n = array.length;

	if (n == 0) {
		return true;
	}
	else {
		return false;
	}
}
these function prints error and warning messages on the console. 
/**
 * Tells whether an array is empty (i.e. contains no element).
 * This function is the same as the previous one, but uses a more compact syntax.
 * @param array Array to examine
 * @return true if array contains no element, false otherwise
 */
function is_empty_array2(array) {
	return array.length == 0;
}
This function is an utility which tells whether an array is empty (i.e. contains no element). 
/**
 * Composes two unary functions into a single one.
 * Recall the meaning of function composition: "(f o g)(x)" (f composed with g applied to x)
 * is equal to "f(g(x))" (f applied to the result of g applied to x).
 * @param f first function to compose
 * @param g second function to compose
 * @return A new function, composition of f and g
 */
function compose(f, g) {
	return function (n) {
		return f(g(n));
	}
}


/**
 * Returns successor of a number.
 * For instance, successor(0) returns 1, successor(41) returns 42.
 * @param n Number
 * @return Successor of n
 */
function successor(n) {
	return n + 1;
}
Successor computes successor of a number. Compose, on the other hand, is a "strange" function: it takes two functions as arguments, and creates a new function which is the equivalent of the first applied to the result of the second. This may seem a bit abstract to the non-mathematical minded, but play an important role. For instance, suppose you want to create "+2", "+3", "+4" functions which add 2, 3 and 4 to their arguments (like the successor): one possibility is to define one new function for each, or you could use the compose abstraction in the following way: 
var plus2 = compose(successor, successor);
What is this? plus2 is a variabile initialized with the result of compose. The latter is given two successor functions as arguments, hence it produces a new function which, when applied to a number n, is equivalent to successor(successor(n)), which is exactly n + 2. Therefore: 
var plus3 = compose(plus2, successor);
var plus4 = compose(plus4, successor);
Are a quick way to accomplish the task.

Observe that last exemple was for didactic purposes: there are better ways to obtain a function which performs a "+N" task. The goal of this example was to show to create, pass and handle functions in JavaScript.
